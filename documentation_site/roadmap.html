Feature Roadmap

Detailed specifications for upcoming features.

1. Real Email Integration

Planned

Why? Currently, reminders are only logged to the console. Users need actual notifications to take action.

Implementation Plan

  • Integrate nodemailer or SendGrid API.
  • Create HTML email templates for "Upcoming Renewal" and "Payment Failed".
  • Update workflow.controller.js to replace console.log with the email service.

2. Smart Dashboard & Analytics

Planned

Why? Users struggle to see the "big picture" of their finances. They need to know their total monthly commitment.

Implementation Plan

  • Create aggregation pipeline in MongoDB to sum prices by user.
  • Add endpoint GET /api/v1/analytics/spend.
  • Return data: { totalMonthly: 120, byCategory: { Entertainment: 50, ... } }.

3. Multi-Currency Support

Planned

Why? Digital nomads and international users have subscriptions in multiple currencies (USD, EUR, etc.).

Implementation Plan

  • Store currency field in Subscription model (already done).
  • Integrate a free API like ExchangeRate-API.
  • Convert all values to the user's "base currency" for the dashboard total.

4. "Cancel for Me" Concierge

Planned

Why? Cancellation processes are often intentionally difficult (Dark Patterns).

Implementation Plan

  • Create a database of "Cancellation Instructions" for popular services.
  • Add a "Cancel" button in the UI that opens a modal with specific steps or a pre-written email template.

5. Shared / Family Subscriptions

Planned

Why? Families and roommates often share costs for services like Netflix or Spotify.

Implementation Plan

  • Add sharedWith array of UserIDs to the Subscription model.
  • Update dashboard to show "My Share" of the cost (e.g., 50% if shared with 1 person).

6. Free Trial Tracker

Planned

Why? Users often forget to cancel free trials before they are charged.

Implementation Plan

  • Add isTrial boolean to Subscription model.
  • Add trialEndDate field.
  • Create a specialized workflow that sends an urgent reminder 24 hours before the trial ends.

7. Bank/Email Import

Planned

Why? Manual entry is a barrier to entry. Automation reduces friction.

Implementation Plan

  • Investigate Plaid API for bank transactions.
  • Alternatively, use Gmail API to scan for keywords like "Receipt", "Subscription", "Invoice".

8. Upcoming Renewals Calendar

Planned

Why? A calendar view helps users visualize their cash flow throughout the month.

Implementation Plan

  • Implement GET /upcoming-renewals.
  • Return subscriptions sorted by renewalDate asc.
  • Frontend: Render a simple calendar UI highlighting payment days.

9. User Profile Management

Planned

Why? Basic account management is currently missing.

Implementation Plan

  • Implement PUT /users/:id to allow updating name and email.
  • Add password reset flow (Request Reset -> Email Link -> New Password).

10. Admin Panel

Planned

Why? System administrators need visibility into platform usage and health.

Implementation Plan

  • Create a new role admin in User model.
  • Create middleware adminMiddleware to protect routes.
  • Build a dashboard showing Total Users, Active Subscriptions, and Error Rates.
